### Interaction Pattern (The "Wonky" Part)

The maintainer repo orchestrates changes to the public repo via `publish-from-dogfood`:
1. Takes `--public-repo /path/to/localsetup-2`
2. Harvests `.cursor/` edits into `_localsetup/` (canonical source)
3. Re-deploys Cursor from source
4. Validates and commits

**Canonical source rule**: `_localsetup/` is source of truth; `.cursor/` is deploy target.

## Key Decisions

### 1. Git History Preservation

**Option A - Fresh repo with grafted history** (RECOMMENDED)
- Create new repo at `localsetup-v2`
- Use `git filter-repo` or manual file copy to create clean structure
- Preserve commit hashes via `git subtree add` or `git rebase --onto`
- Pros: Clean history, no pollution
- Cons: Commit hashes change

**Option B - Use localsetup-2 as base, merge maintainer via subtree**
- Add maintainer as remote, merge via subtree
- Pros: Preserves full history from both
- Cons: Complex git graph, dual remote pointing to different GitHub repos

**Recommendation**: Option A with commit attribution. The user is "switching to different agentics framework" and wants a "clean house." A streamlined repo with clear structure is more valuable than preserving exact commit hashes from two repos that were always meant to be temporary.

### 2. Version Unification

- **Drop maintainer's 2.3.3 versioning** - it's a fork that was out of sync
- **Use localsetup-2's 2.9.5** as the base version
- Maintainer scripts already bump based on conventional commits, so they integrate cleanly

### 3. Content Separation Strategy

| Content | Destination | Rationale |
|---------|-------------|-----------|
| `_localsetup/` (framework) | `localsetup-v2/_localsetup/` | Canonical source - no change |
| `.cursor/` (deploy target) | `localsetup-v2/.cursor/` | Part of framework deployment |
| `.kilocode/` (rules) | `localsetup-v2/.kilocode/` | Migration target - stays |
| `scripts/` (automation) | `localsetup-v2/scripts/` | Consolidated from maintainer |
| `docs/` (public+internal) | `localsetup-v2/docs/` | Merged - some docs adapted |
| `.githooks/` | `localsetup-v2/.githooks/` | From maintainer |
| Publishing workflows | `localsetup-v2/.github/workflows/` | GitHub Actions unified |
| `.github/ISSUE_TEMPLATE/` | `localsetup-v2/.github/` | From localsetup-2 |

### 4. Content to NOT migrate

- `maintainer/audit/` - Generated artifacts, not source
- `maintainer/.agent-tmp/` - Temporary baseline files
- `maintainer/docs/ATTRIBUTION.md` - Merge into main docs
- `maintainer/docs/CURSOR_PACKAGING_AND_SYNC.md` - Merge into REPO_BOUNDARIES.md
- `maintainer/scripts/validate-maintainer-workflow` - Deprecated post-consolidation
